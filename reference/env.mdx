---
title: "zls env: inspect config paths and runtime environment"
description: "Complete reference for the zls env command output: version, config paths, log file location, and how to use it to debug ZLS configuration issues."
---

Running `zls env` prints a JSON object with runtime environment information. Use it to answer questions like "which config file did ZLS load?" or "where is the log file?" without needing to dig through system directories.

```bash
zls env
```

## Example output

```json
{
  "version": "0.16.0-dev",
  "global_cache_dir": "/home/user/.cache/zls",
  "global_config_dir": "/etc/xdg",
  "local_config_dir": "/home/user/.config",
  "config_file": "/home/user/.config/zls/zls.json",
  "log_file": "/home/user/.cache/zls/zls.log"
}
```

## Fields

<ResponseField name="version" type="string">
  The semantic version of ZLS. Release builds use `MAJOR.MINOR.PATCH` (e.g., `"0.15.0"`). Development builds use `MAJOR.MINOR.PATCH-dev.HEIGHT+HASH` (e.g., `"0.16.0-dev.321+ab3c1f2"`).
</ResponseField>

<ResponseField name="global_cache_dir" type="string | null">
  The path to ZLS's cache directory. ZLS stores build runner artifacts and other cached data here. `null` if the platform cache directory cannot be resolved.
</ResponseField>

<ResponseField name="global_config_dir" type="string | null">
  The global (system-wide) configuration directory. On Linux this is typically `/etc/xdg`; on macOS it is `/Library/Application Support`; on Windows it is `%ProgramData%`. ZLS looks here for `zls.json` if no user-level config file is found. `null` if it cannot be resolved.
</ResponseField>

<ResponseField name="local_config_dir" type="string | null">
  The user-specific configuration directory. On Linux this is typically `~/.config`; on macOS `~/Library/Application Support`; on Windows `%APPDATA%`. ZLS looks here first when searching for `zls.json`. `null` if it cannot be resolved.
</ResponseField>

<ResponseField name="config_file" type="string | null">
  The full path to the `zls.json` file that ZLS actually loaded. `null` if no config file was found in any search location. If this is `null` and you expected a config file to be loaded, check that the file exists at `{local_config_dir}/zls/zls.json`.
</ResponseField>

<ResponseField name="log_file" type="string | null">
  The full path to the `zls.log` log file. Normally this is `{global_cache_dir}/zls/zls.log`. `null` if the cache directory is unavailable. You can override this path with the [`--log-file`](/reference/cli#log-file-path) flag.
</ResponseField>

## Common use cases

**Config file not being loaded:**

```bash
zls env | grep config_file
```

If `config_file` is `null`, ZLS did not find a `zls.json` in any of its search paths. Create the file at the path shown by `local_config_dir` (appending `/zls/zls.json`), or pass `--config-path` explicitly.

**Finding the log file:**

```bash
zls env | grep log_file
```

Open the path shown to inspect ZLS log output. You can also override the log location with `--log-file`.

<Tip>Pipe the output through a JSON formatter for easier reading: `zls env | python3 -m json.tool`</Tip>
